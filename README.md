# Carbon Crunch: AI-OCR Receipt Pipeline

## Overview
This repository contains a robust, confidence-aware OCR pipeline capable of extracting structured JSON data from noisy, real-world receipt images. 

The system was designed to handle high levels of variance in receipt quality, including faded ink, dot-matrix printing, wrinkles, shadows, and background color tints. It successfully processed a batch of 369 images, generating field-level confidence scores and calculating a final financial expense summary.

## The Pipeline Architecture
The project is divided into three core modules:

### 1. Image Preprocessing (OpenCV)
Raw images are dynamically standardized before being passed to the OCR engine:
* **Upscaling:** Images are enlarged by 2x using Cubic Interpolation to thicken text structures and prevent fragmentation.
* **Sanitization:** Images are converted to grayscale and softened with a Gaussian Blur to reduce high-frequency noise.
* **Binarization:** Adaptive Thresholding is applied to aggressively eliminate shadows and colored backgrounds while preserving text.

### 2. Text Detection (EasyOCR)
The sanitized image arrays are passed into the EasyOCR deep learning engine. Text blocks are extracted alongside their native mathematical probability scores (0.0 to 1.0) to establish a baseline for system reliability.

### 3. Key Information Extraction & Structuring
Rather than relying purely on brittle Regular Expressions, extraction utilizes a hybrid approach:
* **Total Amount:** Extracted using a Maximum Value Heuristic, which converts all matched price patterns to floats and logically selects the highest valid value.
* **Line Items:** Extracted using a sequential State Machine that retains text lines in memory and dynamically pairs them with the first subsequent price pattern. A "Stop Word" barrier prevents summary metadata (e.g., "Total", "Tax") from being logged as items.
* **Confidence Flagging:** Field-level confidence scores are calculated, and any extracted field scoring below 0.70 is tagged in a `"flags"` array for downstream review.

## Repository Structure
* `notebook.ipynb`: The main Jupyter Notebook containing the full end-to-end pipeline and batch-processing script.
* `/JSON_Outputs/`: A directory containing the 369 individual `.json` files generated by the pipeline.
